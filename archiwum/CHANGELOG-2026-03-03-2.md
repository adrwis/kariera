## Zmiany

### 1. Theme: 23 box-shadows + 8 rgba backgrounds → CSS vars
**Problem:** 31 hardcoded `rgba()` wartości w komponentach nie adaptowały się w dark mode.
**Rozwiązanie:** 11 nowych CSS vars (6 shadows + 5 badges) w `:root` i `[data-theme="dark"]`, zamiana wszystkich 31 wystąpień.
| Plik | Zmiana |
|------|--------|
| `css/style.css` | Nowe: `--kr-focus-glow`, `--kr-dropdown-shadow`, `--kr-popup-shadow`, `--kr-thumb-shadow`, `--kr-glow-secondary-strong`, `--kr-modal-overlay`, `--kr-badge-salary-bg`, `--kr-badge-deficit-bg`, `--kr-badge-balanced-bg`, `--kr-badge-surplus-bg` |

### 2. SEO: description sync + og:image meta
**Problem:** `meta[name="description"]` w index.html nie zgadzało się z `defaultDesc` w app.js. Brak `og:image:width/height/alt`.
**Rozwiązanie:** Sync description + 5 nowych meta tagów (og:image:width/height/alt, twitter:image:alt).
| Plik | Zmiana |
|------|--------|
| `index.html` | Sync description (3 tagi), +og:image:width/height/alt, +twitter:image:alt |

### 3. WCAG: aria-pressed + school dropdown + heading hierarchy
**Problem:** Category chips i sort buttons bez initial `aria-pressed`. School dropdown bez `aria-selected`/`activedescendant`/CSS active. Fallback detail h1→h3→h4 skip.
**Rozwiązanie:** `aria-pressed` na 13 buttonów w HTML. School dropdown: +id, +aria-selected, +aria-activedescendant, +CSS `--active`. Heading fix h3→h2, h4→h3.
| Plik | Zmiana |
|------|--------|
| `index.html` | `aria-pressed="false"` ×10 chips, `aria-pressed` ×3 sort buttons |
| `js/app.js` | School dropdown: id+aria-selected+activedescendant, headings h3→h2/h4→h3 |
| `css/style.css` | `.filters__dropdown-item--active` added to existing hover rule |

### 4. Mobile: font sizes + touch targets + slider thumbs
**Problem:** 13 selektorów z font-size < 14px na mobile. 6 elementów z touch target < 44px. Slider thumbs 18px.
**Rozwiązanie:** Bump do 0.8rem @≤600px. min-height: 44px na checkboxy, popup linki, footer, fallback links. Slider 28px.
| Plik | Zmiana |
|------|--------|
| `css/style.css` | @media ≤600px: font sizes, touch targets, slider thumbs, sort btn fix |

### 5. UX: breadcrumbs + scroll position + loading indicator
**Problem:** Brak breadcrumbs na detail. Scroll position tracony na back-nav. Brak loading state.
**Rozwiązanie:** `nav.breadcrumb` w renderRichDetail+renderFallbackDetail. `scrollPositions` map z isPopstate flag. Disabled buttons podczas loadData().
| Plik | Zmiana |
|------|--------|
| `js/app.js` | Breadcrumbs (2×), scrollPositions+isPopstate, init() loading state |
| `css/style.css` | `.breadcrumb`, `.breadcrumb__list`, `.breadcrumb__item` styles |

### 6. Theme: matchMedia dynamic listener + inputs autocapitalize
**Problem:** Zmiana OS theme nie reagowała. Inputy bez autocapitalize/autocorrect na mobile.
**Rozwiązanie:** `addEventListener('change')` na prefers-color-scheme. `autocapitalize="off" autocorrect="off"` na 3 inputach.
| Plik | Zmiana |
|------|--------|
| `js/app.js` | matchMedia change listener |
| `index.html` | autocapitalize+autocorrect na #searchQuery, #schoolFilter, #resultsSearchInput |
